# DevNexus v0.8.0

## Highlights

- Added the unified MQ Client plugin for RabbitMQ and Kafka.
- Added RabbitMQ connection management, Management API browsing, publish, safe consume preview, message templates, and history.
- Added Kafka connection management, metadata browsing, produce, non-committing consume preview, message templates, and history.
- Added MQ secret encryption and history redaction for passwords, management credentials, SASL secrets, and sensitive headers.

## Safety Boundaries

- RabbitMQ preview consumes messages with requeue by default unless Ack is explicitly selected.
- Kafka preview consumption does not commit offsets.
- The first MQ release does not expose destructive operations such as queue purge/delete, exchange delete, topic create/delete, offset commit, or broker/topic configuration changes.
