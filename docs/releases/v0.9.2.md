# DevNexus v0.9.2

## Highlights

- Consolidates the LAN Chat v0.9.0 and v0.9.1 release line into a single v0.9.2 release note.
- Adds LAN Chat as a bottom-left Chat launcher next to the theme switch instead of a normal left-sidebar plugin entry.
- Adds a global floating chat window that keeps the current Redis/SSH/S3/MQ/API workspace untouched.
- Adds local LAN Chat identity persistence, public-room records, direct conversation records, messages, transfer history, and SQLite-backed Tauri commands.
- Keeps exactly one fixed public room: `public-lobby`, shown as `公共聊天室`, with custom group creation/join removed from the UI and rejected by the backend.
- Merges the public room and direct chats into a single conversation list, using distinct card backgrounds: blue for the public room and green for direct chats.
- Improves public-room UDP delivery by sending both broadcast packets and UDP unicast fallback packets to discovered online devices. Oversized public-room messages automatically fall back to TCP unicast to avoid UDP datagram size limits.
- Adds direct P2P chat with online/offline indicators, offline send blocking, and clear status prompts.
- Reworks all LAN Chat file delivery to use sender-hosted LAN file URLs instead of embedding Base64 in messages. Image, audio, and video previews auto-load from the sender service, and generic files download through the native save-path dialog.
- Adds WeChat-style sender labels, separated message bubbles, automatic scroll-to-latest behavior, and resizable chat window/panes.
- Adds device nickname setup, stable device identity migration, developer logs, status bar integration, unread badges, member list, and transfer cleanup.
- Adds a Windows portable ZIP asset to the tag-triggered GitHub Release alongside the Windows NSIS installer.

## LAN Chat Scope

- Public room messages use UDP broadcast plus discovered-device UDP unicast fallback for better LAN reliability.
- Direct chat remains P2P and uses the discovered or manually entered peer IP/port.
- The public room is mandatory, cannot be deleted, and is the only group-style room in this release.
- Custom group rooms from earlier development snapshots are hidden and no longer supported.
- File transfer uses sender-hosted LAN file URLs. The sender must stay online while receivers preview media or download generic files.
