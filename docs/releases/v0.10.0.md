# DevNexus v0.10.0

## Highlights

- New `Confluence Publisher` plugin: edit local Markdown files in a built-in Monaco editor, render a live Confluence-style preview, and publish/update pages on Confluence Server / Data Center with one click.
- Connection management with site URL, username, and password/token; credentials are encrypted with AES-256-GCM in local SQLite, consistent with other DevNexus plugins.
- Markdown → Confluence Storage Format conversion via `unified` + `remark-parse` + `remark-gfm`, covering headings, bold/italic, inline code, fenced code blocks, links, images, tables, task lists, blockquotes, ordered/unordered lists, and horizontal rules.
- Math support: inline `$...$` and block `$$...$$` (and `latex` / `math` fenced blocks) are converted to Confluence `mathinline` / `mathblock` macros.
- Mermaid diagrams: ` ```mermaid ` blocks are wrapped in an `html` macro that pulls Mermaid via CDN at view time.
- Local image attachments: `![alt](./local.png)` references are uploaded to the page via `cmd_confluence_upload_attachment`, and `ri:url` references are rewritten to `ri:attachment` after upload.
- File-to-page mapping persisted in localStorage so reopening a `.md` file recognizes the linked Confluence page and switches the publish dialog to update mode.
- Backend Rust commands: `cmd_confluence_list_connections`, `cmd_confluence_save_connection`, `cmd_confluence_delete_connection`, `cmd_confluence_test_connection`, `cmd_confluence_list_spaces`, `cmd_confluence_list_pages`, `cmd_confluence_create_page`, `cmd_confluence_update_page`, `cmd_confluence_upload_attachment`.

## Validation

- `npm test`
- `npm run build`
- `cd src-tauri && cargo check`

## Known Limits

- First-version target is Confluence Server / Data Center 7.x+ via REST API v1 with Basic Auth and Personal Access Token (Bearer) auth for SSO environments. Confluence Cloud OAuth is not in scope yet.
- Math and Mermaid rely on Confluence-side macros (`mathinline` / `mathblock` and `html`); spaces without those macros enabled will fall back to literal source.
- Local image upload only follows relative or absolute filesystem paths; remote `http(s)://` and `data:` URLs are left untouched.
