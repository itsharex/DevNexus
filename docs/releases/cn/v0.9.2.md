# DevNexus v0.9.2

## 亮点

- 将 LAN Chat v0.9.0 与 v0.9.1 发布线合并为一个 v0.9.2 发布说明。
- 将 LAN Chat 放到左下角主题切换旁的 Chat 入口，而不是普通左侧插件导航。
- 新增全局悬浮聊天窗口，不影响当前 Redis/SSH/S3/MQ/API 工作区。
- 新增 LAN Chat 本地身份、公共房间、私聊会话、消息、传输历史和 SQLite-backed Tauri 命令。
- 固定保留一个公共房间 `public-lobby`，显示为 `公共聊天室`，移除自定义群聊创建/加入 UI，并在后端拒绝该能力。
- 将公共房间和私聊合并为单一会话列表，用蓝色和绿色卡片背景区分。
- 优化公共房间 UDP 投递，同时发送广播包和已发现设备 UDP 单播补发；超大公共房间消息自动降级为 TCP 单播，规避 UDP 数据报大小限制。
- 新增点对点私聊、在线/离线状态、离线发送阻止和清晰状态提示。
- 重构 LAN Chat 文件传输，改为发送者本地 LAN file URL 承载，不再把 Base64 嵌入消息。图片、音频、视频从发送者服务自动预览，普通文件通过原生保存路径对话框下载。
- 新增类似微信的发送人标签、独立消息气泡、自动滚动到最新消息、可调整聊天窗口和面板尺寸。
- 新增昵称设置、稳定设备身份迁移、开发者日志、状态栏集成、未读角标、成员列表和传输清理。
- Tag 触发 GitHub Release 时，除 Windows NSIS 安装包外额外上传 Windows 绿色版 ZIP。

## LAN Chat 范围

- 公共房间消息使用 UDP 广播加已发现设备 UDP 单播补发，以提升局域网可靠性。
- 私聊保持 P2P，通过发现到或手动输入的 peer IP/port 连接。
- 公共房间是强制存在、不可删除的，也是本版本唯一的群组类房间。
- 早期开发快照里的自定义群房间已隐藏且不再支持。
- 文件传输使用发送者托管的 LAN file URL，接收者预览媒体或下载普通文件时发送者需要保持在线。
