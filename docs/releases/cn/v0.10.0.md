# DevNexus v0.10.0

## 亮点

- 新增 `Confluence Publisher` 插件：在内置 Monaco 编辑器中编辑本地 Markdown 文件，实时渲染类 Confluence 预览，并一键发布/更新到 Confluence Server / Data Center 页面。
- 连接管理支持站点 URL、用户名、密码/Token；凭据与其他 DevNexus 插件保持一致，使用 AES-256-GCM 加密后存储在本地 SQLite。
- Markdown → Confluence Storage Format 转换基于 `unified`、`remark-parse`、`remark-gfm`，覆盖标题、粗体/斜体、行内代码、围栏代码块、链接、图片、表格、任务列表、引用块、有序/无序列表和分割线。
- Math 支持：行内 `$...$`、块级 `$$...$$`，以及 `latex` / `math` 围栏代码块会转换为 Confluence `mathinline` / `mathblock` 宏。
- Mermaid 图表：` ```mermaid ` 块在本地用于预览，发布时渲染为 SVG，封装为 `.drawio` XML 附件上传到 Confluence 页面，并通过 Confluence draw.io 宏嵌入。
- 本地图片附件：`![alt](./local.png)` 引用会通过 `cmd_confluence_upload_attachment` 上传到页面，上传后将 `ri:url` 改写为 `ri:attachment`。
- 文件到页面映射持久化在 localStorage 中，重新打开 `.md` 文件时可识别已关联的 Confluence 页面，并将发布弹窗切换为更新模式。
- Space/page tree 发布：左侧面板加载 Confluence space 和页面层级，可直接选择父页面并将新的 Markdown 内容发布到该页面下。
- 发布历史：本地 SQLite 历史保留已发布 Markdown 快照，可重新打开、修改或删除本地历史记录。
- S3/OSS 兼容性：S3 连接支持手动 Bucket 列表，适配阿里云 OSS 等账号拥有对象权限但没有 `ListBuckets` 权限的场景。
- 文档刷新：新增 `AGENTS.md` 的 agent 指导、`CLAUDE.md` 的 Claude Code 指导，以及 `.qoder/repowiki` 仓库知识库，覆盖架构、插件、数据库、构建、测试、发布和核心插件参考。
- 后端 Rust 命令：`cmd_confluence_list_connections`、`cmd_confluence_save_connection`、`cmd_confluence_delete_connection`、`cmd_confluence_test_connection`、`cmd_confluence_list_spaces`、`cmd_confluence_list_pages`、`cmd_confluence_create_page`、`cmd_confluence_update_page`、`cmd_confluence_upload_attachment`。

## 已知限制

- 首版目标是 Confluence Server / Data Center 7.x+，通过 REST API v1 支持 Basic Auth 和 SSO 环境常用的 Personal Access Token（Bearer）认证。Confluence Cloud OAuth 暂不在范围内。
- Math 依赖 Confluence 侧宏（`mathinline` / `mathblock`）。Mermaid 发布要求目标 Confluence 已安装 draw.io 插件；无效 Mermaid 语法可能导致 draw.io 附件生成失败并留下未解析宏。Confluence 代码宏不支持的语言（如 `http`）会省略 language 参数，避免服务端宏校验错误。
- 本地图片上传只处理相对或绝对文件系统路径；远程 `http(s)://` 和 `data:` URL 保持原样。
- 手动 S3/OSS Bucket 列表要求账号对每个指定 Bucket 具备对象级访问权限；它不会绕过云厂商侧权限控制。
